    BSS maxAlloc (20%) + IL maxAlloc (20%) = 40% combined < 70% cap

  The cap becomes relevant if:
    - MaxAlloc for BSS or IL is increased above 35%

  It prevents a future admin from raising BSS maxAlloc to 50% + IL to 50%,

3.4  BREAK-EVEN ANALYSIS
--------------------------

  Protocol breaks even when:
    Revenue >= Operating costs
    $30,624/year >= $1,140 - $4,140/year

  At minimum opex ($1,140/year):
    Need premium fee = $1,140
    Premium income needed = $1,140 / 3% = $38,000/year
    At U=10%, M=1.063: Annual premium = TVL x 10% x 6.175% x 1.063
    TVL_breakeven = $38,000 / (0.10 x 0.06175 x 1.063) = $578,900

  At full opex ($4,140/year):
    Premium income needed = $4,140 / 3% = $138,000/year
    TVL_breakeven = $138,000 / (0.10 x 0.06175 x 1.063) = $2,101,500

  At realistic U=25%:
    Annual premium = TVL x 25% x 6.175% x 1.156
    TVL_breakeven (min opex) = $38,000 / (0.25 x 0.06175 x 1.156) = $2,131
                               (effectively negligible)
    TVL_breakeven (full opex) = $138,000 / (0.25 x 0.06175 x 1.156) = $7,733

  Summary:
    Break-even TVL is EXTREMELY LOW because operating costs are minimal.
    At any meaningful TVL ($100K+), the protocol is operationally profitable.
    The real question is whether protocol revenue justifies development costs.
